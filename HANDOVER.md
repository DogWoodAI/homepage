# DogWoodAI 홈페이지 — 내수화 인수인계 문서

> 작성일: 2026-06-11
> 목적: 기존 홈페이지 업체(웹호스팅·관리 대행)에서 벗어나 자체 운영(내수화)하기 위한 재구축 사이트의 구조 설명, 운영 방법, 보완 과제 정리.

---

## 1. 프로젝트 개요

- 기존 dogwoodai.com(업체의 PHP/그누보드 기반)을 **순수 정적 사이트(HTML + CSS + JavaScript)** 로 재구축한 것입니다.
- **서버 프로그램, 데이터베이스, 빌드 도구가 전혀 필요 없습니다.** 파일을 웹서버에 올리기만 하면 동작합니다.
- 외부 의존성은 ① Google Fonts(폰트), ② YouTube(영상 임베드) 두 가지뿐이며, 둘 다 무료입니다.
- 한/영 전환(KOR/ENG), 홈 풀페이지 휠 스크롤, 호버 드롭다운, 문의 폼 UI, 팝업, 플로팅 버튼 등 원본 사이트의 기능을 재현했습니다.

### 기술 스택
| 항목 | 내용 |
|---|---|
| 마크업/스타일 | HTML5, CSS3 (단일 파일 `css/style.css`) |
| 동작 | 순수 JavaScript (프레임워크 없음 — jQuery, React 등 미사용) |
| 폰트 | Google Fonts: Poppins, Noto Sans KR (CDN 로드) |
| 영상 | 로컬 mp4 + YouTube iframe 임베드 |
| 서버 요구사항 | **없음** (정적 파일 호스팅이면 충분) |

## 3. 핵심 동작 원리 (담당자가 알아야 할 5가지)

### ① 공통 헤더/푸터는 JS가 만들어 끼워 넣습니다
각 HTML에는 `<div id="site-header"></div>`, `<div id="site-footer"></div>` 자리만 있고,
`js/app.js`가 페이지 로드 시 메뉴·푸터·팝업·플로팅 버튼(CONTACT US/TOP)을 생성해 넣습니다.
→ **메뉴 항목/링크를 바꾸려면 app.js 상단의 `nav` 객체 한 곳만 수정**하면 전 페이지에 반영됩니다.

### ② 한/영 전환 (KOR/ENG)
- 텍스트가 들어갈 자리에 `data-i18n="키이름"` 속성이 붙어 있습니다.
- 실제 문구는 `js/translations.js`의 `en:{...}`, `ko:{...}` 두 사전에 키-값으로 들어 있습니다.
- 분량이 큰 페이지(solution, rndd, projects, approach, ip-papers)는 **해당 HTML 파일 하단의
  `Object.assign(window.I18N.en, {...})` / `(window.I18N.ko, {...})` 블록**에 페이지 전용 문구가 있습니다.
- 선택 언어는 브라우저 localStorage에 저장되어 페이지를 이동해도 유지됩니다.
- **문구 수정 방법: 화면의 문구로 파일 내 검색 → 같은 키의 en/ko 값을 함께 수정.**

### ③ 홈(index.html)의 풀페이지 스크롤
- `js/fullpage.js`가 데스크톱(가로 993px 이상)에서 휠을 한 번 굴리면 한 섹션씩 이동시키고,
  우측 점(도트)으로 현재 위치를 표시합니다. 모바일에서는 자동으로 일반 스크롤이 됩니다.
- 홈 헤더는 히어로(영상) 위에서 투명/흰 로고, 그 아래 섹션부터 흰 배경/검정 로고로 자동 전환됩니다.

### ④ 서브페이지 배너
- 페이지 주소에 따라 `app.js`의 `setBanner()`가 `<body data-banner="0~3">`을 지정하고,
  CSS가 `assets/main/s_visual00~03.jpg` 중 해당 배너 이미지를 깔아줍니다.

### ⑤ 영상
- **YouTube 임베드**: solution.html의 DogWoodPro 섹션 (영상 ID: `egHPLYa-4e4`, `7gUwX3mZw58`).
  다른 영상으로 바꾸려면 iframe의 `embed/영상ID` 부분만 교체.
- **로컬 mp4**: 홈 히어로 배경(ad01.mp4), 홈 글로브 영상(dogwood_01.mp4),
  Business Project 3번(Aveva.mp4)·7번(mp4.mp4). `<video>` 태그라 어디서든 재생됩니다.

| 하고 싶은 일 | 수정 위치 |

| 프로젝트 추가/수정 | `projects.html` 하단 인라인 사전(`pjN.*`)과 `P` 배열, 이미지는 `assets/project/` |

| R&DD 과제 수정 | `rndd.html` 하단 인라인 사전(`rdN.t/p/c`)과 그림 `assets/rndd/` |
| 팝업 교체/중단 | 이미지: `assets/main/popup01.jpg` 교체. 중단: index.html `<body ...>`에 `data-popup="off"` 추가 |

## 6. 실서버 배포 가이드 (내수화 절차)

이 사이트는 정적 파일이므로 **어떤 정적/일반 웹호스팅이든** 올릴 수 있습니다.

### 호스팅 선택지
| 선택지 | 비용 | 비고 |
|---|---|---|
| 국내 웹호스팅 (카페24·가비아 등 일반형) | 약 월 1,100~5,500원 | FTP로 폴더 업로드만 하면 끝. 비개발자에게 가장 익숙. 용량/트래픽 사양만 확인 |
| GitHub Pages | 무료 | 파일당 100MB 제한(현 영상 최대 67MB → OK), 깃 사용법 필요 |
| Netlify | 무료 플랜 | 드래그&드롭 배포 가능, 월 100GB 트래픽 |
| Cloudflare Pages | 무료 | ⚠️ **파일당 25MB 제한 — dogwood_01.mp4(67MB), Aveva.mp4(31MB) 업로드 불가.** 쓰려면 해당 영상을 YouTube 임베드로 교체 필요 |

### 배포 절차 (국내 호스팅 기준 예시)
1. 호스팅 가입 → FTP 접속정보 수령
2. 이 폴더 전체(단, `serve.py`, `.claude/`, `HANDOVER.md` 제외 가능)를 웹 루트에 업로드
3. 도메인(dogwoodai.com)을 새 호스팅으로 연결 — 보통 호스팅사가 안내하는 **네임서버(NS) 변경** 또는 **A레코드 변경**
4. HTTPS 적용 — 요즘 호스팅은 **무료 SSL(Let's Encrypt)을 기본 제공**합니다. 별도 연 8만원 인증서 불필요
5. 메일(@dogwoodai.com) MX 레코드가 끊기지 않도록 DNS 이전 시 기존 MX 값 그대로 복사 (7절 참고)
6. 전환 기간(최소 1~2주)은 기존 업체 서버를 유지한 채 DNS만 새 곳으로 변경 → 문제 시 되돌리기 가능

## 7. ★ 업체에서 받아야 할 것 체크리스트 ★

코드는 못 준다고 해도, 아래는 **회사 자산이므로 반드시 확보**해야 합니다.

### A. 도메인 (가장 중요 — 이것 없으면 내수화 불가)
- [ ] **dogwoodai.com 등록인(소유자)이 누구 명의인지 확인** — 업체 명의라면 회사(주식회사 도그우드에이아이) 명의로 변경 요청
- [ ] 도메인이 등록된 **레지스트라(등록대행사) 이름과 계정 접근 권한** (또는 회사 계정으로 이관)
- [ ] 타 등록기관으로 옮길 경우 **도메인 이관 인증코드(Auth Code/EPP)** 발급
- [ ] **현재 DNS 레코드 전체 내역서** (A, CNAME, **MX**, TXT, SPF/DKIM 등) — 화면 캡처라도 받기

### B. 이메일
- [ ] `dogwoodai@dogwoodai.com` 메일이 **어디서 호스팅되는지** (업체 메일서버인지, 외부 서비스인지)
- [ ] 업체 메일서버라면: **메일함 데이터 백업**(내보내기) + 계정 목록 + 해지 전 이전 일정 협의
- [ ] 이전처: 구글 워크스페이스(약 인당 월 8천원대), 네이버웍스, 가비아 메일 등 → MX 레코드 변경 필요

### C. 콘텐츠/데이터
- [ ] **뉴스 게시판 글 데이터** (그누보드 DB) — 글이 있다면 내보내기(엑셀/DB덤프) 요청
- [ ] **이미지·영상 원본 파일** (특히 고해상도 원본, 인쇄용 로고 AI/PSD가 있다면 함께)
- [ ] **문의 폼 수신 이메일 설정값** (현재 폼 제출이 어느 메일로 가는지)
- [ ] (참고) 웹 콘텐츠 자체는 이미 이 재구축본에 반영되어 있음

### D. 계약/운영
- [ ] **계약서 사본**: 해지 통보 기한, 위약금, **제작물(디자인·콘텐츠)의 저작권 귀속 조항** 확인
- [ ] 해지 희망일 통보 후에도 **DNS 전환 완료까지 1~2개월 병행 유지** 협의
- [ ] 네이버 서치어드바이저/구글 서치콘솔 등록이 업체 계정으로 되어 있는지 → 권한 이양 or 자체 재등록
- [ ] SSL 인증서(연 8만원)는 **갱신 거절** — 새 호스팅의 무료 SSL로 대체

### 비용 비교 (내수화 효과)
| 항목 | 현재(업체) | 내수화 후 |
|---|---|---|
| SSL | 80,000원/년 (VAT별도) | **0원** (Let's Encrypt 무료) |
| 호스팅·관리 | 240,000원/년 (VAT별도) | 0원(무료 플랫폼) ~ 약 66,000원/년(국내 호스팅) |
| 도메인 | (업체 포함 추정) | 약 15,000~25,000원/년 (직접 등록) |
| **합계** | **약 352,000원/년 (VAT포함)** | **약 2~9만원/년** |

※ 메일을 구글 워크스페이스 등으로 옮기면 그 비용은 별도(선택).

## 9. 보완 필요 사항 (우선순위순)

### 🔴 오픈 전 필수
1. **문의 폼 실제 전송 기능 없음** — 현재는 화면 동작(글자수 카운트, 완료 토스트)만 구현.
   해결 선택지(코딩 최소화 순):
   - **Formspree / Tally / EmailJS** 같은 폼 수신 서비스 연결(무료 플랜 있음) — form 태그에 action 한 줄 추가 수준
   - 호스팅사 제공 폼메일(CGI) 사용
   - 임시방편: 제출 버튼을 `mailto:dogwoodai@dogwoodai.com` 링크로 대체
2. **개인정보처리방침·이용약관 페이지 없음** — 푸터 링크가 빈 링크(#) 상태.
   문의 폼으로 개인정보(이름·이메일)를 수집하므로 **개인정보보호법상 처리방침 게시가 필요**합니다.
   기존 사이트의 방침 전문을 받아 `privacy.html`, `terms.html`로 추가하고 푸터 링크 연결.
3. **`home.html` 등 참고용 파일 삭제 후 배포** — 재구축 과정에서 받아둔 원본 참고 사본이 남아 있다면 배포본에서 제외.

### 🟡 권장
4. **구 URL 리다이렉트** — 기존 `…/theme/dogwoodai/….php` 주소들이 검색엔진에 색인되어 있음.
   같은 도메인으로 새 구조를 올리면 옛 주소는 404가 됨 → 호스팅의 리다이렉트 기능(.htaccess 등)으로
   주요 페이지만이라도 새 주소로 301 연결 권장 (예: `aisolution_00.php → /solution.html`).
5. **SEO 마무리**: 페이지별 `<meta name="description">` 점검, OG 태그(공유 미리보기), `sitemap.xml`,
   `robots.txt` 추가, 네이버 서치어드바이저·구글 서치콘솔 재등록.
6. **파비콘 정식 세트** — 현재 PNG 로고를 임시 사용 중. 원본 favicon 파일 세트를 받아 교체 권장.
7. **뉴스 운영 방식 결정** — 현재는 HTML 직접 수정 방식. 게시글이 잦다면 추후 무료 헤드리스 CMS나
   간단한 빌드 도구 도입 검토 (당분간은 4절 가이드대로 수동 추가로 충분).
8. **대용량 영상 최적화** — dogwood_01.mp4(67MB)는 페이지에서 클릭 시에만 로드되도록 해두었지만,
   트래픽 절약을 원하면 YouTube 임베드로 교체 가능(이미 유튜브 채널에 동일 영상 존재).

### 🟢 선택

10. 한/영 별도 URL(SEO용 다국어 구조)은 현재 토글 방식(원본 사이트와 동일)으로 충분하나, 영문 SEO를 키우려면 추후 검토
